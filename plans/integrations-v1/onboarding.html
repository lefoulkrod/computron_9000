Part IV
MCP subprocess flows
MCP plugins are brokers too. Same UID (1001), same vault socket, same verification pattern. The wizard collects the config the server needs, vault-seals it, and spawns uvx or npx under systemd-style supervision.

09 · Custom MCP · advanced
The escape hatch — run any MCP server the user can reach with uvx / npx. Args come in as a JSON array; env-from-vault lets the user mark specific keys as secret without typing them into plaintext fields.

Custom MCP server

You’re on your own here — but anything that speaks MCP-stdio is fair game.

Secret env vars (vault-sealed)
NOTION_API_KEY secret
secret_xxxxxx… · stored encrypted
You’re installing third-party code. Computron will fetch and run this package under a non-privileged user, but it will still have network access and whatever capabilities the server itself implements. Only install servers you trust.
